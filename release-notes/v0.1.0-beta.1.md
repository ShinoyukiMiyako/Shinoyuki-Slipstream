## v0.1.0-beta.1 — 首个公开测试版（Forge 1.20.1 网络层流量优化）

Slipstream 是 Shinoyuki 优化系列的网络分支：BetterAutoSave 之于存盘/加载 IO，Slipstream 之于网络出站带宽。本版是首个公开版本，包含一整套出站流量遥测与可回退的带宽优化，全部已在内部生产服验证。系列遵循同一条优先级：数据安全 > 稳定性 > 兼容性 > 性能——任何需要牺牲数据保真、服务器稳定或客户端兼容来换带宽的手段，一律不做。

### 本版能力

- 出站流量遥测：服务端 clientbound 流量分析，纯测量不改协议。按包类型给出数量、压缩前/线上字节、压缩比、出站占比；按玩家给出热力分解；统计同一区块为多玩家重复序列化的冗余。命令 `/slipstream report | reset | capture`，快照与原始抓包落 `logs/slipstream/`
- zstd 线上压缩：按连接协商，单端可回退。对端没装或没开 Slipstream 时自动退回原版 zlib，原版客户端照常连接、线上字节与原版一致。区块包线上字节约削减 15-35%（默认级别 3，可调 1-19）
- 大包支持：抬高原版包体积上限，使地图数据、载具/结构 NBT、蓝图粘贴等大包不再崩溃或被拒，内置替代 XLPackets / PacketFixer
- serialize-once 广播去重：同一次同步广播中每个区块包只压缩一次，其余接收者复用同一压缩帧，跳过重复的 Deflater 开销
- 小包聚合（战斗场景）：把成片的微型实体状态包（motion / move / head-rotate）在一个窗口内合并成批，消除逐包封帧开销，并把整批交给 zstd。内部生产服密集战斗实测出站约削减三分之一。窗口默认 20ms
- L2 实体字段 delta（战斗场景）：把高频实体状态包按每实体每字段重编码为 zigzag-varint 增量，无损。在聚合之上再叠加，内部实测实体流量再降约 18%
- requireClient 强制双端：开启后服务端在登录期拒绝未启用全部协商优化的客户端，供大型服务器保证全员启用优化。默认关闭

### 安全与兼容

- 全部处理在 netty event-loop 上完成，从不触碰服务器主线程（tick 线程），不增加 MSPT、不影响 TPS
- 所有协议改写无损：实体字段 delta 有逐字节往返单元测试门禁；入站 zstd 帧有解压炸弹大小防护
- 单端可回退零兼容代价：原版及未安装本 mod 的客户端照常连接，所有协商类优化对其退回原版行为
- 协商类功能（zstd / 聚合 / L2）需双端均安装且开启同一选项方才生效；其能力通道在启动时注册，开关后需重启服务端（及客户端）

### 为什么是 beta

- 首个公开版本，核心功能已在内部生产服内测验证，但公开多服样本尚少
- 协商类与战斗场景优化（聚合 / L2）默认保守关闭，仅 zstd 默认开启；稳态行为仍在持续观察

### 配置与命令

- 完整开关与命令见 README。关键开关：`zstdEnabled`（默认开）、`aggregateEnabled`（默认关）、`l2Enabled`（默认关）、`requireClient`（默认关）
- 配置文件 `config/Shinoyuki-Optimize/slipstream/common.toml`

### 升级与安装

- drop-in：放进 `mods/` 即可，单端即可获得遥测与 zstd 单端回退行为
- 协商类功能（zstd / 聚合 / L2 / 大包）需双端同装同开，改动这些开关后两端均需重启
